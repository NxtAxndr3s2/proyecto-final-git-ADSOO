**Que es Git en 8 lineas** :

Git es un sistema de control de versiones distribuido.

Permite registrar los cambios realizados en archivos de un proyecto.

Facilita trabajar en equipo sin sobrescribir trabajo ajeno.

Guarda el historial completo del proyecto mediante commits.

Permite crear ramas para desarrollar nuevas funciones sin afectar la versión principal.

Facilita regresar a versiones anteriores si ocurre un error.

Sincroniza proyectos con plataformas como GitHub o GitLab.

Es una herramienta esencial para el desarrollo de software moderno.

**10 Comandos de Git Explicados**

git init: Inicializa un repositorio Git en una carpeta.

git add .: Agrega todos los archivos modificados al área de preparación.

git status: Muestra el estado actual del repositorio.

git commit -m "mensaje": Guarda los cambios agregados con un mensaje descriptivo.

git push: Envía los commits al repositorio remoto.

git pull: Obtiene la versión más reciente del repositorio remoto y la mezcla con la local.

git clone URL: Clona un repositorio remoto en tu computadora.

git branch: Muestra las ramas existentes.

git checkout -b nombre_rama: Crea y cambia a una nueva rama.

git merge rama: Fusiona una rama con la rama actual.

**Flujo de Trabajo Recomendado**

Clonar el repositorio o actualizarlo con git pull.

Crear una nueva rama para tu tarea con git checkout -b nombre_rama.

Realizar y guardar cambios en los archivos.

Agregar los cambios con git add.

Crear un commit con git commit -m "Mensaje claro".

Enviar la rama al remoto con git push.

Crear un Pull Request (PR) para revisión.

Revisar, aprobar y fusionar los cambios.
